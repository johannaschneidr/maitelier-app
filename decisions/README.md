# /decisions

This folder contains technical context for AI agents working in this codebase.

Files here record architecture constraints, naming conventions, and key technical decisions — not how the code works (read the code for that), but *why* it works that way and what must not change without deliberate intent.

**DECISIONS.md** — the primary file. Read it before suggesting architectural changes, adding dependencies, or proposing new patterns.

Rules for updating:
- Add to Active Constraints only when a decision has been explicitly confirmed as a hard constraint.
- Add to Out of Scope when something has been considered and rejected — so it doesn't resurface.
- Keep entries to one line. This is a reference, not a document.
